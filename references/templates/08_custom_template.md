# 自定义模板

完全自定义的 Skill 结构，适用于未涵盖的特殊需求场景。

## 使用场景

以下情况适合使用自定义模板：

- 独特的业务流程
- 特殊的技术需求
- 混合多种功能类型
- 实验性功能开发
- 特定领域的专业工具

## 模板结构

```markdown
---
name: [skill-name]
description: [一句话描述]
license: LICENSE-CC-BY-NC-SA 4.0 in LICENSE.txt
author: [作者名]
---

[功能描述]

[功能描述]

[步骤说明]

## 自定义配置

### 选择现有模板组合

如果需求部分匹配，可以组合现有模板：

```
基础框架：通用工具类
+ 扩展功能：文档处理类
+ 扩展功能：浏览器操作类
```

### 混合模板示例

```markdown
---
name: web-scraper-pro
description: 高级网页数据采集工具
author: 开发者名
---

# Web Scraper Pro

## 简介

结合浏览器操作和内容提取功能的专业数据采集工具。

## 核心功能

1. 浏览器导航与交互
2. 动态内容处理
3. 数据提取与解析
4. 数据导出

## 工作流

[混合工作流程说明]
```

## 模板定制建议

### 1. 功能规划

- 明确核心功能
- 定义边界范围
- 规划扩展方向

### 2. 结构设计

- 模块化设计
- 清晰的职责划分
- 合理的依赖关系

### 3. 接口设计

- 统一的输入格式
- 标准的输出格式
- 清晰的错误处理

### 4. 文档编写

- 详细的使用说明
- 清晰的示例
- 完整的 API 文档

## 提交自定义模板

如果您创建了高质量的自定义模板，欢迎贡献到本项目：

1. 在 `references/templates/` 目录下创建新模板
2. 遵循命名规范：`XX_template_name.md`
3. 包含完整的模板说明
4. 提供使用示例
```

## 自定义模板检查清单

在创建自定义模板前，请确认：

- [ ] 需求是否可以用现有模板满足？
- [ ] 核心功能是否清晰定义？
- [ ] 工作流程是否合理？
- [ ] 输入输出是否明确？
- [ ] 错误处理是否完善？
- [ ] 文档是否完整？
```
