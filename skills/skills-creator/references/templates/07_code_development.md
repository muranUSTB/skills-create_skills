# 代码开发类 Skill 模板

适用于代码生成、重构、审查、调试、文档生成等软件开发场景。

## 模板结构

```markdown
---
name: [skill-name]
description: [一句话描述]
license: LICENSE-CC-BY-NC-SA 4.0 in LICENSE.txt
author: [作者名]
---

# Skill Name

## 简介

[详细介绍代码开发的功能]

## 支持的语言

### 编程语言
- JavaScript/TypeScript
- Python
- Java
- Go
- Rust
- [其他语言]

### 技术栈
- Web 前端
- Web 后端
- 移动开发
- 嵌入式开发
- [其他领域]

## 功能特性

### 1. 代码生成

代码生成功能：
- 根据需求生成代码
- 模板代码生成
- 脚手架生成
- 单元测试生成

### 2. 代码审查

代码审查功能：
- 代码质量分析
- 安全漏洞检测
- 性能优化建议
- 最佳实践检查

### 3. 代码重构

重构功能：
- 代码简化
- 模式应用
- 架构优化
- 技术债务清理

### 4. 代码调试

调试功能：
- 错误分析
- 日志分析
- 性能分析
- 逻辑追踪

### 5. 文档生成

文档功能：
- API 文档生成
- 注释生成
- README 生成
- 变更日志生成

## 工作流

### Step 1：理解需求

分析用户需求：
- 理解编程语言
- 了解技术栈
- 明确功能要求

### Step 2：代码生成/分析

执行相应操作：
- 生成代码
- 分析代码
- 审查代码
- 调试问题

### Step 3：质量检查

执行质量检查：
- 语法检查
- 风格检查
- 单元测试
- 集成测试

### Step 4：输出结果

返回处理结果：
- 代码文件
- 审查报告
- 优化建议
- 文档内容

## 使用示例

```markdown
请用 Python 实现一个快速排序算法。

请审查「src/utils.js」的代码质量。

请帮我在「user_service.py」中添加分页查询功能。

请为「api/routes」目录生成 API 文档。
```

## 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| language | string | 是 | 编程语言 |
| operation | string | 是 | 操作类型 |
| source | string | 是 | 代码来源 |
| requirements | object | 否 | 需求描述 |

## 输出格式

| 输出类型 | 格式 | 说明 |
|---------|------|------|
| 代码 | 代码文件 | 生成的代码 |
| 报告 | Markdown | 审查报告 |
| 文档 | Markdown | API 文档 |

## 注意事项

- 遵循项目编码规范
- 保持代码可读性
- 考虑性能影响
- 包含适当的注释
- 编写测试用例

## 最佳实践

- SOLID 原则
- DRY 原则
- KISS 原则
- YAGNI 原则
- 清晰的命名
```
